## Technical Implementation Notes

### Components Needed:
1. `CallButton` - Audio/Video call initiation buttons
2. `IncomingCallModal` - Full-screen incoming call UI
3. `OutgoingCallModal` - Calling screen
4. `ActiveCallView` - Main call interface
5. `CallControls` - Mute, camera, end call buttons
6. `VideoGrid` - Layout for multiple participants
7. `ScreenShareView` - Screen sharing display
8. `CallNotification` - In-app call alerts
9. `CallHistory` - Call log in chat
10. `PiPWindow` - Picture-in-Picture floating window

### WebRTC Flow:
1. User clicks call button
2. Send call signal via WebSocket
3. Recipient receives notification
4. Accept/Decline call
5. Establish WebRTC peer connection
6. Exchange ICE candidates
7. Stream audio/video
8. Handle call controls (mute, camera, etc.)
9. End call and cleanup

### Backend Requirements:
- WebRTC signaling server
- TURN/STUN servers for NAT traversal
- Call state management
- Call history storage
- Recording service (optional)
- Quality monitoring

### Technologies:
- **WebRTC** - Peer-to-peer audio/video
- **Socket.io** - Signaling
- **MediaStream API** - Camera/microphone access
- **Screen Capture API** - Screen sharing
- **Picture-in-Picture API** - Minimized calls
